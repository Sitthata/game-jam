**lever.gd** — Add `_initial_is_on` storage and `reset()`:

```gdscript
var _initial_is_on: bool = false

func _ready() -> void:
    _initial_is_on = is_on        # capture before any toggle
    add_to_group("resettable")
    _update_visual()
    toggled.emit.call_deferred(is_on)

func reset() -> void:
    is_on = _initial_is_on
    _update_visual()
    toggled.emit(is_on)
```

`_initial_is_on` is captured from the `@export var is_on` value set in the editor, which is the designer's intended starting state. `reset()` emits `toggled` so any door listening to this lever also resets its state via signal.

**door.gd** — Add `_initial_open` storage and `reset()`:

The existing `_ready()` always starts the door open (collision disabled). To support doors that start closed, capture the actual initial state from the collision shape rather than hardcoding. Since `_ready()` always calls `_collision.set_deferred("disabled", true)`, door always starts open — so `_initial_open = true` is always correct. Still use the variable pattern for future flexibility.

```gdscript
var _initial_open: bool = true

func _ready() -> void:
    _initial_open = true          # door always starts open per current design
    add_to_group("resettable")
    _collision.set_deferred("disabled", true)
    _sprite.animation = "open"
    _sprite.frame = 0

func reset() -> void:
    set_open(_initial_open)
```

NOTE: door.gd's reset delegates to `set_open()` which already handles collision and sprite correctly. No duplication needed.

**Part B — Add R key handler to `_unhandled_input` in main.gd:**

```gdscript
func _unhandled_input(event: InputEvent) -> void:
    if event.is_action_pressed("timeline_swap"):
        var destination = past if in_present else present
        if _has_collision_at_player(destination):
            print("can't jump")
            return
        in_present = !in_present
        _apply_timeline_state()
        if _is_void_at_player():
            _do_fall_and_respawn()
    elif event.is_action_pressed("reset_room"):
        _reset_room()

func _reset_room() -> void:
    for node in get_tree().get_nodes_in_group("resettable"):
        if node.has_method("reset"):
            node.reset()
```

Key design points:
- `_reset_room()` is a named function (not inline lambda) for readability and future extension (Phase 3 may need to call it programmatically)
- `has_method("reset")` guard avoids crashes if any node is in the group without a reset method
- Player position is NOT reset (per D-discretion from CONTEXT.md — no player teleport on R press)
- Timeline state is NOT reset — R only resets object positions/states, not which timeline is active
